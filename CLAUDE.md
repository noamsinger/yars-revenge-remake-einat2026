# Yars' Revenge — Claude Code guide

## Build & run

```
mvn javafx:run      # run in dev
mvn javafx:run -Djavafx.args="--debug"   # run with debug logging
mvn compile         # compile only
mvn package         # fat JAR
mvn versions:use-latest-releases         # update dependency versions in-place
```

## Architecture

```
app/        YarsRevengeApp (main + --debug flag), SceneManager, LogDirDefiner
model/      GameState, GameConstants, entity/, wave/
controller/ InputHandler, GameController, AutoPilot
engine/     GameLoop (60 Hz fixed timestep), CollisionDetector
renderer/   GameRenderer, sub-renderers, RenderContext (snapshot), sprites
audio/      PcmSynthesizer, AudioManager, SoundEffect enum
screen/     Screen interface + all screens
config/     GameConfig
```

## Key constraints

- **Audio**: `AudioClip` must load from temp WAV files — data: URIs are not supported. All sounds are generated by `PcmSynthesizer.generate(SoundEffect)` and written to a temp directory at startup. New sounds: add to `SoundEffect` enum, add a case in `PcmSynthesizer.generate()`, register duration in `AudioManager.clipDurationMs` if it loops.
- **Rendering**: 1920×1080 logical space. `GameRenderer` applies `gc.scale()` to map to physical window. All coordinates are logical.
- **Missile audio**: every code path that kills a Swirl must queue `STOP_SWIRL_LOOP`, or the loop plays forever and blocks `RESUME_BG_HUM`. Also reset `lastQMode` in `GameLoop` on wave transitions to prevent spurious `RESUME_BG_HUM`.
- **Shield modes** cycle per wave via `WaveFactory`: `ARCH_BARRICADE → CYCLING_FENCE → ROTATING_CIRCLE → RANDOM_SWARM → repeat`.
  - `ARCH_BARRICADE`: polar arc around Quotile, left half only. `Shield.rebuild()` + `Shield.update()`.
  - `CYCLING_FENCE`: flat rectangular grid, snaking conveyor (alternating row directions). `Shield.rebuildRect()` + `Shield.updateScrolling()`.
  - `ROTATING_CIRCLE`: full orbiting circle (all angles, cells go off-screen). `Shield.rebuildRotating()` + `Shield.updateRotating()`.
  - `RANDOM_SWARM`: random cloud in polar arc zone, 3× cell count, follows Quotile Y. `Shield.rebuildSwarm()` + `Shield.updateSwarm()`.
- **Wave config**: `WaveFactory.forWave(n)` returns `WaveConfig` with a `ShieldMode` field. `scrollingShield()` returns true only for `CYCLING_FENCE`.
- **Logging**: SLF4J + Logback. Audio lifecycle events at DEBUG, perf warnings at WARN. Pass `--debug` to promote `com.yarsrevenge` to DEBUG. Logs written to `~/Library/Logs/yars-revenge/` (macOS), 7-day retention.
- **Dependency versions**: keep at latest stable release. Run `mvn versions:use-latest-releases` to update. JavaFX pinned to 21.x (must match JDK LTS target); maven-compiler-plugin pinned to 3.x (4.x still beta). Reasons documented in pom.xml comments.

## Entity/update/render pattern

1. Add entity class extending `GameEntity` with `update(dt, state)`.
2. Reference it in `GameState`.
3. Update it in `GameLoop.tick()`.
4. Render it in `EntityRenderer` or a new sub-renderer.
5. Handle collisions in `CollisionDetector`.

## AutoPilot algorithm (Shift-A to toggle)

Priority order — higher priorities override lower ones:

1. **Dodge threats** (torpedo within 250px, swirl warning/in-flight, own cannon within 200px Y): evaluate 8 compass directions, score by distance-from-danger minus shield-cell penalty, always move (never zero velocity). Resets nibble timer.
2. **Fire cannon** (charged + nibbled ≥ 3s): align Y to Quotile from current X position, fire.
3. **Nibble shield cells**: move to nearest alive cell. Accumulates nibble timer.
4. **Touch Quotile** (no cells, not charged): move to Quotile body to charge.
5. **Patrol** (cannon in flight, nothing else to do): drift to left-of-NZ center.
