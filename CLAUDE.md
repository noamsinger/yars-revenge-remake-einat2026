# Yars' Revenge — Claude Code guide

## Build & run

```
mvn javafx:run      # run in dev
mvn compile         # compile only
mvn package         # fat JAR
```

## Architecture

```
app/        YarsRevengeApp (main), SceneManager
model/      GameState, GameConstants, entity/, wave/
controller/ InputHandler, GameController
engine/     GameLoop (60 Hz fixed timestep), CollisionDetector
renderer/   GameRenderer, sub-renderers, RenderContext (snapshot), sprites
audio/      PcmSynthesizer, AudioManager, SoundEffect enum
screen/     Screen interface + all screens
```

## Key constraints

- **Audio**: `AudioClip` must load from temp WAV files — data: URIs are not supported. All sounds are generated by `PcmSynthesizer.generate(SoundEffect)` and written to a temp directory at startup. New sounds: add to `SoundEffect` enum, add a case in `PcmSynthesizer.generate()`, register duration in `AudioManager.clipDurationMs` if it loops.
- **Rendering**: 1920×1080 logical space. `GameRenderer` applies `gc.scale()` to map to physical window. All coordinates are logical.
- **Missile audio**: every code path that kills a `QuotileMissile` must queue `STOP_MISSILE_LOOP`, or the loop plays forever and blocks `RESUME_BG_HUM`.
- **Shield (even waves)**: uses `Shield.rebuildRect()` — flat rectangular grid. Dimensions are scaled from `SHIELD_COLS` (×1.3 width) and `shieldRows` (×1.2 height). Cells are square (`CELL_W × CELL_W`). Wall follows Quotile Y via offset in `updateScrolling()`.
- **Shield (odd waves)**: uses `Shield.rebuild()` — polar arc around Quotile. Updates via `Shield.update()`.
- **Wave config**: `WaveFactory.forWave(n)` returns `WaveConfig`. `scrollingShield=true` on even waves.

## Entity/update/render pattern

1. Add entity class extending `GameEntity` with `update(dt, state)`.
2. Reference it in `GameState`.
3. Update it in `GameLoop.tick()`.
4. Render it in `EntityRenderer` or a new sub-renderer.
5. Handle collisions in `CollisionDetector`.
